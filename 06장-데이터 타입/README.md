# 06장. 데이터 타입

`데이터 타입` 은 `값의 종류` 를 나타내기 위한 방법 입니다.

`숫자 1` 과 `문자 "1"` 을 `타입` 으로 구분하는 이유는, 각 타입별 연산과 처리 방법에 필요하기 때문입니다.

<br />

`타입` 은 `총 7가지` 이며, 크게 나누면 `2가지` 로 구분할 수 있습니다.

* `객체 타입`
* `원시 타입`
  * `number`
  * `string`
  * `boolean`
  * `undefined`
  * `null`
  * `symbol`



<br /><hr /><br />



# 1. 숫자 타입

자바스크립트의 `number` 타입은 `부동 소수점` 형식 입니다.

그리고 `진수` 에 대한 구분도 하지 않기 때문에, 모든 `number` 타입은 `10진수` 로 해석하는 특징이 있습니다.

<br />

특정 숫자값을 표현하는 `유한한 값` 이외에도 다음과 같은 특별한 값을 표현할 수도 있습니다.

* `Infinite`: 양의 무한대
* `-Infinite`: 음의 무한대
* `NaN`: 숫자가 아님을 표현



<br /><hr /><br />



# 2. 문자열 타입

`string` 타입은 텍스트 데이터를 표현 합니다.

자바스크립트에서 `string` 타입은 `원시 타입` 에 속하기 때문에, `불변성` 을 가지는 `리터럴` 값 입니다.



<br /><hr /><br />



# 3. 템플릿 리터럴

`string` 타입 데이터를 나타내기 위한 방법중 하나 입니다.

일반적인 `string` 타입 값은 `"값"` 또는 `'값'` 의 형식으로 나타냅니다.

`템플릿 리터럴` 은 기존의 `string` 과는 다르게, `런타임` 에서 `string` 으로 변환되는 처리방식을 가집니다.

<br />

다음은 `템플릿 리터럴` 의 특징 입니다.

* 멀티 라인 입력

```javascript
const value = `
  Hello,
  World
`;
```

<br />

* 표현식 삽입

```javascript
const name = "KIM";
const value = `Hello, ${name}`;
```

<br />

* 태그드 템플릿

```javascript
/**
 * @param { string } strings
 * @param { string } bgColor
 */
const styleMixins = (strings, bgColor) => {
  const styles = strings[0];
  const ending = strings[1].trim() || ";";

  const color = 
    (bgColor === "#000")
      ? "#fff" 
      : (bgColor === "#fff")
        ? "#000"
        : "inherit";

  return `
    ${styles} ${bgColor}${ending}
    color: ${color};
  `.replace(/\s{2,}/g, "\n  ");
};

const styleInstance = styleMixins`
  font-size: 20px;
  font-weight: 900;
  background-color: ${"#000"};
`;
```



<br /><hr /><br />



# 4. 불리언 타입

`불리언 타입` 은 논리값인 `true` 와 `false` 두가지 값을 나타냅니다.



<br /><hr /><br />



# 5. undefined 타입

자바스크립트에서 `값이 없음` 을 나타내는 타입으로 `undefined` 값 자체가 타입 입니다.

변수의 선언만 할 경우, `undefined` 로 초기화 됩니다.

이는 변수 선언으로 확보한 메모리에 잔존하는 `쓰레기 값 (Garbage Value)` 를 지우기 위함 합니다.



<br /><hr /><br />



# 6. null 타입

`null` 역시 `값이 없음` 을 나타내는 타입이며, `null` 값 자체가 타입 입니다.

이는 개발자가 의도적으로 명시하는 `값이 없음` 이며, `undefined` 와는 다른 의미를 갖게 됩니다.

* `undefined` 인 변수: 초기화 되지 않은 변수를 의미
* `null` 인 변수: 개발자가 의도적으로 값을 제거하였다는 의미

<br />

변수에 `null` 을 넣게되면, `참조` 중이었던 메모리를 더이상 `참조하지 않겠다` 라는 의미도 갖게 됩니다.

이렇게 `참조가 끊긴 메모리` 는 `가비지 콜렉터 (Garbage Collector)` 에 의해, `메모리 해제 (Memory Free)` 를 실행하게 됩니다.

<br />

마지막으로 `함수` 가 `유효한 값을 반환할 수 없는 경우` 에도 `null` 을 반환하여 의미를 담을 수 있습니다.



<br /><hr /><br />



# 7. 심벌 타입

`Symbol` 타입은 `변경 불가능한 원시 타입` 값 입니다.

`Symbol` 값은 어떠한 값과도 중복되지 않는 `유일무이` 한 값 입니다.

그래서 객체의 `Property Key` 를 만들기 위해 사용합니다.

<br />

`Symbol` 을 생성하는 방법은 다른 타입들과는 다르게, `Symbol()` 함수를 사용하여 만들 수 있습니다.

```typescript
const key = Symbol("myKey");

type MyObj = {
  [key]?: string;
};

const myObj: MyObj = {};
myObj[key] = "Hello World";

console.log(myObj[key]); // "Hello World"
```



<br /><hr /><br />



# 8. 객체 타입

자바스크립트는 `Prototype 기반` 언어 입니다.

`Prototype` 이라는 객체가 자바스크립트의 거의 모든 부분을 이루며, 지금까지 살펴본 `원시 타입` 과 구분하는 이유 입니다.

<br />

`객체 타입` 과 `원시 타입` 의 가장 큰 차이는 메모리에 저장된 데이터 형식 입니다.

`원시 타입` 은 오직 하나의 `타입` 을 의미하지만, `객체 타입` 은 많은 타입들을 `묶는 형태` 입니다.



<br /><hr /><br />



# 9. 데이터 타입의 필요성

## 9-1. 데이터 타입에 의한 메모리 공간의 확보와 참조

값은 `저장` 과 `참조` 를 목적으로 합니다.

즉, 값을 훼손하지 않고 `저장` 하고, 다시 `참조` 할 수 있어야 합니다.

<br />

값을 올바르게 저장하기 위해서는, 값이 잘리지 않게 저장할 수 있는 공간이 필요하고, 이 공간을 무작정 크게 만들수도 없습니다.

그래서 특정 타입의 변수를 선언하게 되면, 메모리의 `낭비` 와 `손실` 이 없는 최적의 공간을 확보하게 됩니다.

<br />

변수에 저장된 값을 `참조` 할 경우에도 `참조 주소` 로 부터 `얼마의 메모리 (크기)` 를 읽어야 온전한 값을 읽을 수 있는지가 필요합니다.

이 역시 `타입` 에 따라 정해진 `메모리 크기` 를 기준으로 읽으면 해결되므로, `데이터 타입` 은 `메모리 확보와 참조` 를 위한 필수 요소 입니다.



<br /><hr /><br />



## 9-2. 데이터 타입에 의한 값의 해석

2진수 값 `0b 0100 0001` 은 `숫자 타입` 과 `문자열 타입` 의 해석이 다릅니다.

* 숫자 타입: 65
* 문자열 타입: "A"

<br />

이는 저장된 값을 어떻게 해석할 것인가를 결정해 줍니다.



<br /><hr /><br />



## 9-3. 타입이 필요한 이유 정리

* 변수 선언 시, 최적의 `메모리 확보` 를 위해
* 저장된 값을 참조할 때, `온전한 값` 을 읽기 위해
* 2진수 값의 `해석 방법` 을 결정하기 위해

# 10. 동적 타이핑

## 10-1. 동적 타입 언어와 정적 타입 언어

자바스크립트는 `동적 타입 언어` 입니다.

이는 변수 자체가 타입을 가지는 것이 아닌, 변수에 저장된 `데이터의 타입` 에 의해 결정되는 방식 입니다.

그래서 변수의 값을 재할당 할 때 타입이 달라진다면, 해당 변수의 타입도 달라지게 됩니다.



<br /><hr /><br />



## 10-2. 동적 타입 언어와 변수

동적 타입 언어는 변수 자체의 타입이 없기 때문에, 동일한 변수에 원하는 타입의 데이터를 유동적으로 할당할 수 있습니다.

이는 매우 편리하지만, 트레이드 오프로써 복잡한 프로그램일수록 `데이터의 값을 추적하기 어렵다` 는 단점을 가집니다.

즉, `동적 타입 언어` 는 `유연성` 은 높지만 `신뢰성` 이 떨어지는 특징을 가집니다.
